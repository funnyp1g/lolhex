## 📝 技术要点

### DNS绕过的正确方法

**错误方法**（不生效）:
```javascript
// ❌ 修改全局 dns.lookup 不影响 axios
dns.lookup = function(...) { ... }
```

**正确方法**（唯一可靠）:
```javascript
// ✅ 直接使用IP + Host header
https.request({
  hostname: '84.17.37.217',  // 真实IP
  headers: {
    'Host': 'data.v2.iesdev.com'  // 原域名
  },
  rejectUnauthorized: false  // 禁用证书验证
})
```

### 为什么这个方法有效？

1. **绕过DNS解析**: 直接使用IP地址，不经过DNS查询
2. **Host header**: 告诉服务器我们想访问哪个域名
3. **证书验证**: 因为用的是IP，证书域名不匹配，必须禁用验证

### 安全性说明

⚠️ **禁用证书验证的安全风险**:
- 仅用于特定场景（DNS污染绕过）
- 仅用于可信API（iesdev是Blitz.gg官方API）
- 不适用于敏感数据传输

✅ **安全措施**:
- 仅在云函数内部使用
- 不暴露给小程序客户端
- API返回的是公开数据，不涉及敏感信息

### 常见错误

| 错误 | 原因 | 解决方法 |
|------|------|---------|
| `ECONNREFUSED` | IP已失效 | 更新真实IP（nslookup） |
| `超时` | 网络问题 | 增加timeout，或分批同步 |
| `JSON解析失败` | 数据格式变化 | 更新parseIesdevResponse |
| `成功但数据为空` | 参数错误 | 检查champion_id是否有效 |
